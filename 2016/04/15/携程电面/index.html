Hashtable和HashMap的区别？

  • 并发操作：Hashtable使用同步机制，HashMap没有。
  • fast fail支持：Hashtable和HashMap都可以用Iterator遍历，支持fast fail。而使用Enumeration遍历Hashtable时不支持。(fast fail是指遍历过程中如果另一个线程进行put,remove等操作时会抛出ConcurrentModificationException)
  • hash算法不一样。HashMap充分利用hashcode的每一位，hash值分布更均匀。HashMap内部数组的长度总是2的n次方，可以使散列更加高效。

java.util.concurrent包里面有哪些常用的类？

BlockingQueue

一个支持并发的队列，类似于操作系统课上讲的生产者消费者模型。如果一个thread向满的queue中add或从空的queue中读都会阻塞。
这只是一个接口，具体的实现由ArrayBlockingQueue，DelayQueue等。

ConcurrentMap

支持并发处理的map，也是一个接口。具体实现有ConcurrentHashMap。

CountDownLatch

支持并发。给定一个初始值count，有两个函数await()和countDown()。countDown()会将count值减一，await()会阻塞当前线程直到count=0。

CyclicBarrier

一个线程同步的机制。在一段代码中加入cyclicBarrier.await()，只有所有线程都执行到这一步才能继续。还可以传入一个Runnable，在所有线程抵达barrier之后执行。

Semaphore

类似于操作系统上讲的Semaphore。有acquire(), release()方法，对应于Up, Down操作。

ExecutorService

java中线程池的接口。实现类有ThreadPoolExecutor等。有多种线程池，可以用Executors工厂类获得，newSingleThreadExecutor(), newFixedThreadPool()… 调用execute(Runnable)方法执行一个runnable。

Http status code

  • 502: bad gateway，通常指从服务器收到了错误响应
  • 500: InternalServiceError，服务器发生了错误
  • 401: Unauthorized，没有在请求加token之类的，导致身份验证失败
  • 400: BadRequest，通常是请求中某些字段不符合要求
  • 302,301: 都是重定向到Location指定的URL，一个是临时移动一个是永久移动

ThreadLocal的实现原理？

ThreadLocal是为每个线程维护一个变量副本，每个线程只能读写它本地的那个副本，对其他线程没有影响。
其实很简单里面就是一个Map，key为线程对象，值为对应该线程的变量副本。

关于JVM了解哪些？

只要回答些基础的就可以了。
首先.java文件被编译成.class文件，里面是字节码(byteCode)，是一种可以被JVM执行的指令。
由于JVM实现了程序和操作系统的分离，从而实现了java平台无关性。
JVM是一个虚拟机器，通过软件模拟了处理器，堆栈，寄存器等硬件，有自己的指令系统。
程序执行过程：类加载器加载类->链接->初始化类->执行main()函数。

TCP建立连接和关闭连接的过程？

建立连接三次握手。客户端发送SYN，进入SYNSENT状态，服务器端收到SYN，发送SYNACK，进入SYNRCVD状态。客户端收到SYNACK，进入ESTABLISHED状态，发送ACK。最后服务器端收到ACK，也进入ESTABLISTED，连接建立。

断开连接四次握手。客户端发送FIN，服务器端ACK。服务器端再发送FIN，客户端ACK，结束。
